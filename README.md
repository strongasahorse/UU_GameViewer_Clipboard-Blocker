# UU远程控制剪切板故障修复工具

解决 UU 远程（GameViewer）与微信输入法剪贴板同步功能冲突，导致剪贴板被锁定无法使用的问题。

> ## ⚠️ 重要：本工具的使用场景
>
> **本工具仅适用于同时满足以下条件的情况：**
>
> 1. **使用 UU 远程控制** 进行远程连接
> 2. **使用 微信输入法** 并开启了 **「剪贴板同步」** 功能
> 3. 出现了 **剪贴板被锁定、无法复制粘贴** 的问题
>
> ---
>
> **如不使用微信输入法（或关闭剪贴板同步），UU 远程控制本身不会锁定剪贴板，无需使用本工具。**

## 问题背景

### 冲突原因

UU 远程控制的 `GameViewer.exe` 进程会创建一个隐藏的 `clipboard` 窗口，持续调用 `OpenClipboard` 持有不释放，导致所有其他进程无法访问剪贴板（`OpenClipboard` 返回 FALSE）。

但经过实测发现：**UU 远程控制本身并不会锁定剪贴板**，只有在同时使用 **微信输入法** 并开启了微信输入法的 **「剪贴板同步」** 功能时，才会触发此冲突。

### 冲突原理

微信输入法的剪贴板同步功能会持续监控系统剪贴板变化，而 UU 远程控制出于远程剪贴板共享的需要，也会操作剪贴板。两个软件同时争夺剪贴板访问权时，微信输入法导致 GameViewer 触发了某种竞争状态，使 GameViewer 调用 `OpenClipboard` 后不再释放，形成永久锁定。

如果不使用微信输入法（或关闭其剪贴板同步功能），此问题不会出现。

### 使用场景

本工具**仅适用于**同时满足以下条件的用户：

1. 使用 **UU 远程控制** 进行远程连接
2. 使用 **微信输入法** 并开启了 **剪贴板同步** 功能
3. 出现了剪贴板被锁定、无法复制粘贴的问题

## 解决方案

**核心思路**：既然冲突的根源是 GameViewer 在竞争状态下锁定剪贴板不放，那么直接阻止 GameViewer 使用剪贴板即可。剪贴板同步功能交由微信输入法独立完成，不受影响。

### 具体方法：API Hook 内存修补

在 GameViewer 进程内存中将 `OpenClipboard` 函数入口修补为 `return FALSE`，使其每次调用都立即失败，从根本上阻止其锁定剪贴板。

- ✅ **不杀进程** — GameViewer 正常运行，远程连接不受影响
- ✅ **内存级修补** — 不写磁盘，重启 GameViewer 后自动恢复
- ✅ **自动阻断新进程** — Monitor 模式自动发现并修补新启动的 GameViewer
- ✅ **微信输入法独立工作** — 剪贴板同步功能正常使用

## 使用方法

### 第1步：下载全部脚本

点击仓库页面的绿色 **「Code」** 按钮 → **「Download ZIP」**，解压到本地任意文件夹。
> 所有脚本文件必须放在同一个目录下，不可单独下载单个文件。

### 第2步：选择版本运行

根据你的需求选择对应版本：

| 版本 | 启动文件 | 操作方式 |
|------|---------|---------|
| **版本3 — 有UI阻断版 ✨（推荐）** | 双击 `UI剪切板监控工具_阻断版.bat` | 自动提权管理员，弹出 GUI 窗口，自动阻断 GameViewer，**无感操作** |
| **版本2 — 无UI阻断版** | 双击 `GameViewer剪贴板阻断_无UI版.bat` | 自动提权管理员，黑窗口运行，自动阻断 GameViewer，**无感操作** |
| **版本1 — 监控版** | 双击 `UI剪切板监控工具_监控版.bat` | 隐藏窗口后台运行，仅监控，需手动杀进程（**会中断远程连接**） |

> ⚠️ 首次运行会弹出 **用户账户控制（UAC）** 窗口，点击「是」授予管理员权限即可。

### 开机自启动

运行版本3 后，在 GUI 窗口中勾选 **「开机自启动」** 即可随系统启动自动运行。

### 手动运行（命令行）

.\UI剪切板监控工具_阻断版.bat

## 技术原理

```
OpenClipboard 原始代码              修补后
┌──────────────────────┐          ┌──────────────────────┐
│  mov [rsp+8], rbx    │          │  xor eax, eax        │
│  mov [rsp+10h], rsi  │   ──→   │  ret                  │
│  push rdi            │          │   (后续字节被跳过)     │
│  sub rsp, 30h        │          └──────────────────────┘
│  ... (正常逻辑)       │          return FALSE
│  return TRUE          │
└──────────────────────┘
```

修补字节（x64）：`33 C0 C3`（`xor eax, eax; ret`）

## 内核对象

- `ClipUtil` — 剪贴板检测（`GetOpenClipboardWindow` 等）
- `ClipboardHacker` — API Hook 注入（`WriteProcessMemory` + `VirtualProtectEx`）

## 环境要求

- Windows 10 / 11（64 位）
- 管理员权限（`WriteProcessMemory` 需要）
- 64 位 PowerShell（默认）
